他にも多くの機能があるが、まずは`blocks --live`の表示をコンパクトにしたい。

### 対応方法

簡易的な対処法として、`blocks --live`と同様の情報を表示するプログラムやスクリプトを作成することが考えられる。`blocks --live --json`でJSONデータとしてレポートを受け取り、そのデータを元に表組みして表示する。

---

## ccusage-compact.mjs 仕様

### 概要

`npx ccusage@latest blocks --json` のJSON出力を受け取り、80文字幅のコンパクトなテーブルで表示するNode.jsスクリプト。外部依存なし。

### 実行方法

```bash
node ccusage-compact.mjs                       # 1回実行して終了
node ccusage-compact.mjs --live                # 30秒ごとに自動更新
node ccusage-compact.mjs --live --interval 60  # 60秒ごとに自動更新
```

live時は `\x1b[2J\x1b[H` で画面クリアしてから再描画。Ctrl+Cで終了。

### テーブル設計（合計80文字）

カラム幅: 14 + 10 + 9 + 7 + 7 + 14 = 61文字 + 罫線19文字 = 80文字

| カラム | 幅 | 寄せ | 内容 |
|--------|-----|------|------|
| Date (JST) | 14 | 左 | `MM-DD HH:mm` 形式（UTC→JST変換） |
| Status | 10 | 左 | `35m` / `2h7m` / `ACTIVE` / `gap 3h` / `gap 1h13m` |
| Tokens | 9 | 右 | `438k` / `2.5M` / `-` |
| % | 7 | 右 | `17.4%` / `-`（全ブロック最大totalTokensを上限として計算） |
| Cost | 7 | 右 | `$0.18` / `-` |
| Models | 14 | 左 | 略称をカンマ区切り |

### JSONフィールドマッピング（ccusage実装から確認済み）

ccusage の `blocks --json` が返す各ブロックのフィールド:

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `startTime` | ISO文字列 | ブロック開始（UTC） |
| `endTime` | ISO文字列 | ブロック終了予定（UTC） |
| `actualEndTime` | ISO文字列\|null | 最終エントリ時刻（UTC） |
| `isActive` | boolean | 現在のアクティブブロック |
| `isGap` | boolean | セッション間の空き時間 |
| `totalTokens` | number | 合計トークン数 |
| `costUSD` | number | コスト（USD） |
| `models` | string[] | 使用モデル名リスト |
| `burnRate` | object\|null | `tokensPerMinute`, `costPerHour` |
| `projection` | object\|null | `totalTokens`, `totalCost`, `remainingMinutes` |

- `durationMinutes` フィールドは**存在しない**。`startTime` と `actualEndTime` の差分を計算する
- gap行の期間は `startTime` と `endTime` の差分
- トークン上限は `blocks --json` に含まれないため、全ブロック最大 `totalTokens` で推定

### REMAINING / PROJECTED 行

activeブロックが存在する場合のみ表示:
- **REMAINING**: `tokenLimit - activeBlock.totalTokens`（tokenLimitは全ブロック最大totalTokensで推定）
- **PROJECTED**: `activeBlock.projection.totalTokens` / `projection.totalCost`

### 注意事項

- `wc -L` や `awk length()` はUTF-8罫線文字のバイト数を数えるため幅が大きく出る（正確にはNode.js の `str.length` = 80文字が正しい）
- ccusageはアクティブブロック内にも短いgap行を生成することがある（正常動作）

## 開発ルール

### 更新履歴

- バグ修正・機能追加・変更を行ったら `CHANGELOG.en.md` と `CHANGELOG.ja.md` を両方更新する
- エントリ形式: 日付 → `Added` / `Fixed` / `Changed` / `Removed`（英語）/ `追加` / `修正` / `変更` / `削除`（日本語）のカテゴリ → 内容
- GitHub Issue に対応する場合は `([#N](URL))` 形式で参照を付ける

### ドキュメント

- `README.md` を変更した場合は `README.ja.md` も同期する（日英両方を常に最新に保つ）
- `CHANGELOG.en.md` を変更した場合は `CHANGELOG.ja.md` も同期する（同上）

### バージョン管理

セマンティックバージョニング（`vMAJOR.MINOR.PATCH`）で管理する:
- MAJOR: 後方互換性のない変更（出力フォーマットの大幅変更など）
- MINOR: 機能追加（新オプションなど）
- PATCH: バグ修正

現在の最新バージョン: `v0.2.0`（`0.x` = 安定版宣言前）
